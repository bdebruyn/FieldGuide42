# Subsystems

## Purpose

This directory contains subsystem-specific requirements and design documents.

## Subsystem Directory Template

Each subsystem should use this structure when created:

```text
subsystem-name/
   requirements.md
   design-notes.md
   interfaces.md
   tests.md
   codex-context.md
```

## Subsystem Requirements Template

Each `requirements.md` file should include:

```markdown
# Subsystem Name Requirements

## Purpose

## Scope

## Out of Scope

## Responsibilities

## Functional Requirements

## Non-Functional Requirements

## Data Owned by This Subsystem

## External Dependencies

## Interfaces Consumed

## Interfaces Provided

## Business Rules

## Error Handling

## Security Rules

## Observability Requirements

## Test Requirements

## Open Questions

## Decomposition Candidates
```
