# Architecture Overview

## Purpose

This document describes the current architectural understanding of the Construction Software project.

## Current Architecture Status

Not yet selected.

## Architectural Approach

The architecture will be discovered progressively from decomposed requirements.

## Current Guidance

- Avoid premature architectural commitment.
- Identify domains and subsystem boundaries before implementation.
- Prefer explicit data ownership.
- Prefer explicit interface contracts.
- Prefer testable requirements before implementation.
- Record material architectural decisions as ADRs.
