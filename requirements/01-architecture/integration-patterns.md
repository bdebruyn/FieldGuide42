# Integration Patterns

## Purpose

This document records approved integration patterns between subsystems and external systems.

## Current Status

No integration patterns have been approved yet.

## Candidate Patterns

- Internal API calls
- Domain events
- File import/export
- Database views
- External webhooks
- Batch synchronization

## Rules

- Integration boundaries should be documented as interface contracts.
- Cross-subsystem data writes require explicit approval.
- Error behavior must be documented for each interface.
