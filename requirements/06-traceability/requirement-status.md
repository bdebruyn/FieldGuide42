# Requirement Status

## Purpose

This document tracks requirement status across the project.

## Status Definitions

| Status | Meaning |
|---|---|
| Draft | Captured but not reviewed. |
| Needs Review | Requires user or stakeholder review. |
| Approved | Accepted as a valid requirement. |
| Decomposed | Split into child requirements or subsystem-specific requirements. |
| Ready for Implementation | Clear enough for Codex implementation. |
| In Implementation | Currently being implemented. |
| Implemented | Implementation exists. |
| Tested | Implementation has passing tests. |
| Deferred | Intentionally delayed. |
| Rejected | Not accepted. |
| Superseded | Replaced by another requirement. |
