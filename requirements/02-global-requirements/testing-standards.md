# Testing Standards

## Purpose

This document defines testing expectations.

## General Rules

- Every Codex implementation task should include relevant tests.
- Tests should map to requirement IDs where practical.
- Acceptance criteria should be testable.
- Regression tests should be added for defect fixes.
- Tests should avoid relying on unrelated subsystem behavior unless explicitly required.

## Test Categories

- Unit tests
- Integration tests
- Contract tests
- End-to-end tests
- Security tests
- Migration tests
- Regression tests
