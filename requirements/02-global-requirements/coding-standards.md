# Coding Standards

## Purpose

This document records implementation standards for Codex-generated software.

## Current Status

Language-specific standards are not yet finalized.

## General Rules

- Prefer clear names.
- Prefer small units of behavior.
- Prefer tests with every implementation task.
- Do not modify unrelated subsystems.
- Do not introduce external dependencies without justification.
- Document any material architectural assumption in an ADR.
