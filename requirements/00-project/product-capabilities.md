# Product Capabilities

## Purpose

This document captures broad product capabilities before they are decomposed into formal requirements, subsystems, interface contracts, and Codex implementation tasks.

Capabilities in this file are not automatically approved requirements. They are candidates for decomposition.

## Capability Status Values

| Status | Meaning |
|---|---|
| Candidate | Captured but not reviewed. |
| Needs Clarification | Requires more detail before decomposition. |
| Ready For Decomposition | Clear enough to split into requirements or subsystem ownership. |
| Decomposed | Converted into formal requirements or subsystem documents. |
| Deferred | Intentionally delayed. |
| Rejected | Not part of the project scope. |
