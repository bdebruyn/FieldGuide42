# Glossary

## Purpose

This document defines project terminology.

## Terms

| Term | Definition | Status |
|---|---|---|
| Requirement | A durable statement of expected system behavior, constraint, or quality. | Draft |
| Subsystem | A coherent portion of the system with defined responsibilities and boundaries. | Draft |
| Interface Contract | A documented boundary between subsystems or external systems. | Draft |
| Codex Task Packet | A bounded instruction set given to Codex for implementation. | Draft |
| Traceability | The ability to connect requirements to tasks, implementation, and tests. | Draft |
