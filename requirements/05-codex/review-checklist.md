# Codex Review Checklist

## Purpose

Use this checklist to review Codex-generated changes.

## Checklist

- [ ] The implementation references the correct task.
- [ ] The implementation satisfies the listed requirement IDs.
- [ ] The implementation does not modify unrelated subsystems.
- [ ] The implementation follows applicable coding standards.
- [ ] The implementation follows error-handling expectations.
- [ ] Tests were added or updated.
- [ ] Tests map to acceptance criteria.
- [ ] Interface contracts were updated if behavior changed.
- [ ] Traceability documents were updated.
- [ ] No undocumented dependency was introduced.
- [ ] No architectural assumption was introduced without an ADR.
